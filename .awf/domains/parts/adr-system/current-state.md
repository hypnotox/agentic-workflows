## Current state

Decisions live as append-only ADRs under `docs/decisions/`, parsed by `internal/adr` (a shared `internal/frontmatter` parser) into a generated `ACTIVE.md` status index, regenerated by `awf sync` and drift-checked by a regenerate-and-compare path. Each ADR declares `domains:`, which drives this doc's generated `## Decisions` index (same regenerate-and-compare mechanism). The workflow chain (brainstorm → ADR → plan → resync → impl → review) is enforced by the `awf-*` skills and reviewer agents. The ADR guide (`README.md`), the skeleton (`template.md`), and a plan-authoring guide (`plans/README.md`) are awf-managed singletons (ADR-0021, ADR-0020), so the `.layout.adrReadme` / `.layout.adrTemplate` / `.layout.plansReadme` references resolve in any adopter project. `ACTIVE.md` always renders (a placeholder index when no ADRs exist) so its document-map link resolves out of the box (ADR-0020, partial-item supersedence of ADR-0005/ADR-0006). A new ADR is scaffolded via `awf new adr "<title>"` (ADR-0042), which computes the next sequential number and copies `template.md` with its marker comments stripped and its date/title filled in, rather than by hand; `awf-proposing-adr` invokes it as the first authoring step. ADR-0103 revives the long-dropped `tags:` and `related:` frontmatter: `internal/adr` now lifts both into `adr.ADR`, `awf check` resolves each ADR's `related:` numbers against the decisions dir (`invariant: adr-related-link-resolved`, the ADR analogue of the pitfall/plan link checks), and the corpus is tagged from a governed vocabulary, the relevance currency a later slice will spend in `awf context`.
