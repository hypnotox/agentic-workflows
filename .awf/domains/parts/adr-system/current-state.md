## Current state

Decisions live as append-only ADRs under `docs/decisions/`, parsed by `internal/adr` (a shared `internal/frontmatter` parser) into a generated `ACTIVE.md` status index — regenerated by `awf sync` and drift-checked by a regenerate-and-compare path. Each ADR declares `domains:`, which drives this doc's generated `## Decisions` index (same regenerate-and-compare mechanism). The workflow chain (brainstorm → ADR → plan → resync → impl → review) is enforced by the `awf-*` skills and reviewer agents. The ADR guide (`README.md`), the skeleton (`template.md`), and a plan-authoring guide (`plans/README.md`) are awf-managed singletons (ADR-0021, ADR-0020), so the `.layout.adrReadme` / `.layout.adrTemplate` / `.layout.plansReadme` references resolve in any adopter project. `ACTIVE.md` always renders — a placeholder index when no ADRs exist — so its document-map link resolves out of the box (ADR-0020, partial-item supersedence of ADR-0005/ADR-0006).
