## Current state

Decisions live as append-only ADRs under `docs/decisions/`, parsed by `internal/adr` (a shared `internal/frontmatter` parser) into a generated `ACTIVE.md` status index, regenerated by `awf sync` and drift-checked by a regenerate-and-compare path. Append-only protects rationale, not orthography (ADR-0115): an Implemented ADR's arguments, decision items, alternatives, and consequences are never rewritten, but a meaning-preserving punctuation fix to its `# ` heading line is normalization rather than revision, and is permitted with explicit maintainer approval. ADR-0118 widened that carve-out, partially superseding ADR-0115 Decision item 5: retroactive normalization reaches the full authored body of every ADR and plan, not the heading alone, and the whole corpus was swept to plain punctuation in one act (roughly 7200 sites across 182 files, fences included). Two exemptions stand, both named rather than inferred: ADR-0113, the ADR about the em-dash, by maintainer designation; and prose where the glyph is the subject of its own sentence, which punctuating would make false. What licenses a sweep that large over settled rationale is that word preservation is proven rather than promised: a word-stream comparison (alphanumeric runs only, old versus new) must report zero delta, so the risk is a flat punctuation choice, never lost argument. Each ADR declares `domains:`, which drives this doc's generated `## Decisions` index (same regenerate-and-compare mechanism). The workflow chain (brainstorm → ADR → plan → resync → impl → review) is enforced by the `awf-*` skills and reviewer agents. The ADR guide (`README.md`), the skeleton (`template.md`), and a plan-authoring guide (`plans/README.md`) are awf-managed singletons (ADR-0021, ADR-0020), so the `.layout.adrReadme` / `.layout.adrTemplate` / `.layout.plansReadme` references resolve in any adopter project. `ACTIVE.md` always renders (a placeholder index when no ADRs exist) so its document-map link resolves out of the box (ADR-0020, partial-item supersedence of ADR-0005/ADR-0006). A new ADR is scaffolded via `awf new adr "<title>"` (ADR-0042), which computes the next sequential number and copies `template.md` with its marker comments stripped and its date/title filled in, rather than by hand; `awf-proposing-adr` invokes it as the first authoring step. ADR-0103 revives the long-dropped `tags:` and `related:` frontmatter: `internal/adr` now lifts both into `adr.ADR`, `awf check` resolves each ADR's `related:` numbers against the decisions dir (`invariant: adr-related-link-resolved`, the ADR analogue of the pitfall/plan link checks), and the corpus is tagged from a governed vocabulary, the relevance currency a later slice will spend in `awf context`. Partial-item supersedence (successor carries `related: [predecessor]`, predecessor's status stays live, successor's prose cites the overridden items) additionally owes a **back-pointer**: the successor's number appended to the amended ADR's `related:`, without which the amended Decision item reads as current guidance. That rule had lived only in the pitfalls doc, and 2026-07-15 sweeps measured at least 19 partial amendments with only 7 conformant (a floor: the detectors are verb-anchored, and ADR-0116's Context records an earlier, narrower 6-of-16 measurement); ADR-0116 states it in the `awf-adr-lifecycle` procedure and the `adr-reviewer`'s doc-currency lens, and deliberately declines a mechanical check until there is evidence about whether a stated procedure holds. The same ADR reconciles the skill's append-only statements, which had said a live ADR is status-only and so forbade the very edit the rule requires: `status` **and** cross-reference metadata (`superseded_by:`, `related:`) are editable in place, on the principle that append-only protects rationale, not bookkeeping. The corpus is knowingly left non-conformant (ADR-0116 Decision 6), so a fresh sweep finds residue, not recurrence.
