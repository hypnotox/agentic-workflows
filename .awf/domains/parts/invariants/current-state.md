## Current state

Each machine-enforceable ADR Invariant bullet is declared in one of two forms (ADR-0105): a backed `invariant: <slug>` — which requires a proof `<marker> invariant: <slug>` comment on a **test** — or an `unbacked-invariant: <slug>` carrying a `**Verify:**` note, a reasoned contract deliberately exempt from the proof requirement. A proof marker backs a slug only in a file matching the project's `invariants.testGlobs` (`**/*_test.go` here, so backing means an executed test line); when `testGlobs` is empty the checker falls back to source-glob scope (`invariants.sources`), so the test-scoped teeth are opt-in. A separate advisory `touches-invariant: <slug> — <note>` marker records a production site that relates to an invariant — it never backs a slug and never fails `awf check`. The checker is language-agnostic (anchored path globs matched against slash-separated repo-relative paths per ADR-0077 — `**/*.go` for any depth, `internal/**` to scope a subtree — plus a literal marker) and enforces symmetrically: for an Implemented ADR, a backed slug with no proof marker in scope, an `unbacked-invariant:` slug that nonetheless carries a proof marker, and an `unbacked-invariant:` slug missing its `Verify:` note each fail `awf check`; a marker naming an undeclared slug and a note-less `touches-invariant:` marker are non-failing advisories. Backing is opt-in per bullet; untagged bullets remain textual contracts. Enforcement has a retirement escape (ADR-0031): a successor Implemented ADR can declare `retires_invariants: [<slug>]` in its frontmatter to drop a slug from enforcement when the change removes its backing code, leaving the declaring ADR's record intact; a retirement naming a slug no Implemented ADR declares is a dangling retirement and fails `awf check`. ADR-0104 exposes the one-to-one Implemented-ADR `slug → declaring-ADR` map (duplicate-refusing, retirement-applying) as `invariants.DeclaringADRs`, shared by `Check` and `awf context`'s Tier-1 relevance join — which ADR-0106 makes a two-marker union scan over `invariants.sources` and `invariants.testGlobs` (recognising both the proof `invariant:` and the `touches-invariant:` marker), labelling each governing invariant backed or unbacked.
