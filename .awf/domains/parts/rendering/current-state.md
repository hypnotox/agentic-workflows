## Current state

The render engine is a marker-section overlay (`<!-- awf:section -->`) layered onto Go `text/template` with `missingkey=zero`, guarded by a hard publication-safety check that rejects any unresolved-variable placeholder. Convention parts are raw input (ADR-0034): only awf-owned template defaults are templated — each part slot is replaced by a brace-free sentinel before execution and the raw body restored verbatim afterwards, so a part renders byte-for-byte (a literal `{{` is never parsed) and is never variable-interpolated. Doc paths are awf-given via the `.layout` namespace (never `.vars`); skills cite docs through `.layout.docs.*`/`.layout.workflowRef`, and a doc-gated skill is suppressed when its doc is disabled. Catalog docs ship static default content with a per-doc section taxonomy; domain docs are the one data-driven exception, injecting a generated index as forced body. Always-on neutral singletons render the agent guide, the two ADR-system files (`README.md`, `template.md`), a plan-authoring guide (`plans/README.md`), and — since ADR-0043 promoted them from toggleable docs — the workflow, documentation-standard, and agent-guide-authoring-standard docs, each suppressible with a `local: true` sidecar. The six non-agent-guide singletons render and validate through one shared `internal/project/singleton.go` table (`plainSingletons`) driving the existing `renderKind` primitive, rather than a hand-rolled loop duplicated between rendering and validation (ADR-0021, ADR-0020, ADR-0043). An optional `.awf/bootstrap.sh` neutral config-tree singleton renders once when enabled (ADR-0040, relocated off the repo root by ADR-0047), pinned to the rendering binary's `project.Version`, and is excluded from the dead-reference scan. The `hooks` singleton (ADR-0048) likewise renders three inert git-hook payload scripts under `.awf/hooks/` as a unit when enabled — var-driven with graceful fallbacks and a pin-aware bootstrap shim, no overridable sections, excluded from the managed-markdown scans, and labeled per script in unset-var advisories; awf never activates them. `awf check` also scans every managed rendered markdown file for inline links whose file-relative target is missing on disk and fails on a `dead-reference` drift; the link extractor is the pure `internal/refs` package (ADR-0020). Per-kind dispatch across the render and check paths (enable array, catalog pool, declared sections, output path, labels) resolves through one ordered kind-descriptor table whose facets are accessor functions, absorbing the catalog map-vs-slice and adapter-vs-neutral path asymmetries in a single place (ADR-0027). `RenderAll` splits its passes (ADR-0037): neutral artifacts (docs, `AGENTS.md`, domain docs, the ADR-system singletons) render once, while adapter artifacts (skills, agents, and an optional `@AGENTS.md` bridge) render once per enabled adapter `Target`, the same body and frontmatter written to each runtime's path. A project enables adapters via the `targets` array (default `[claude]`), resolved against a registry holding `claudeTarget` and a bridge-less `cursorTarget` (Cursor reads `AGENTS.md` natively); the path-keyed lock carries one entry per target with no hash collision, and Sync prunes a dropped target's whole directory tree. Catalog specs carry per-artifact default `data:` merged under sidecars at sidecar resolution (present key wins, even null — ADR-0045), every var/data interpolation degrades to coherent generic prose when unset (the extended publication-safety contract), and the render context exposes the effective skills set as `.skills`; `awf check` prints non-failing unset-var notes and fails on a `dead-skill-reference` drift — a rendered `<prefix>-<name>` token naming a known skill outside the effective set (ADR-0046). The render context also exposes `.commitScopes` — the display-formatted `audit.allowedScopes` list, empty when accept-any — folded into the config hash of referencing artifacts like `.skills`, so an `audit.allowedScopes` edit reflags exactly the templates that quote it (ADR-0051).
