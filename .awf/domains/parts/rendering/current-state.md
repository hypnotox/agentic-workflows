ADR-0146 makes Pi a five-output target: two subagent files, one handoff file, and dashboard index and protocol files. The protocol is derived from the Go-owned telemetry descriptor with declared input attribution and hashing; only the dashboard index consumes widget display configuration. The three extension factories exchange bounded versioned observations and active-branch association data, while the dashboard owns confined resident writes, provisional and structured-resume identity, the closed lifecycle-enforcing workflow loader, controlled canonical refresh, widget and overlay UI, confirmed maintenance, and shutdown drain. Pi publishes one discoverable semantic router, fixed hidden governed bodies, and separately discoverable reviewer agents; non-Pi targets retain ordinary skill discovery. Missing or incompatible canonical binaries leave visible stale or degraded state rather than local aggregation or diagnosis. ADR-0150 makes an enabled bootstrap authoritative, then permits PATH failure or refusal to fall through only to an advertised repository `dashboard-awf-path` command. The launcher receives the absolute project root in `AWF_DASHBOARD_PROJECT_ROOT`; dual failure preserves both bounded causes, and one successful launcher is captured for the Pi session so ref advancement affects only new sessions. The generic rendered runner remains repository-neutral and omits the two development commands.

The topic producer renders each valid pair to `<docsDir>/topics/<domain>/<topic>.md`, emits a title-and-summary-sorted `<docsDir>/topics/<domain>/index.md`, and adds compact topic navigation to the owning domain page in place of a per-domain ADR index. Authored topic Markdown uses the raw-part convention and authoring-comment strip rather than YAML interpolation. Topic pages and indexes are ordinary managed Markdown output-plan nodes, so manifest hashing, brownfield backup, drift, and prune share the existing lifecycle with no topic-specific lock field. Together with the generated decision index `INDEX.md`, they are the active authority surface that replaced the retired `ACTIVE.md` and per-domain ADR indexes; the one-time project-atomic cutover deleted those legacy outputs through the recoverable upgrade transaction.

ADR-0145 made every rendered workflow checkpoint an ordered durable-and-visible protocol, ADR-0149 made the memory file optional with an exact active `Effort: <id>`, and ADR-0152 splits the boundary semantics into two rendered protocols. ADR-0127 changes what the `reviewing-impl` skill renders: it now passes the session range it already derives (`${baseSha}..${headSha}`) to `awf audit` instead of auditing "the branch", so the terminal review audits the session it just reviewed under any branching strategy.

The render engine is a marker-section overlay (`<!-- awf:section -->`) layered onto Go `text/template` with `missingkey=zero`, guarded by a hard publication-safety check that rejects any unresolved-variable placeholder. Convention parts are raw input (ADR-0034): only awf-owned template defaults are templated: each part slot is replaced by a brace-free sentinel before execution and the raw body restored verbatim afterwards, so a part renders byte-for-byte (a literal `{{` is never parsed) and is never variable-interpolated. ADR-0121 adds the one removal to the substitutions: whole-line `awf:comment` authoring comments are stripped from part bodies (and template sources) at ingestion, retiring `parts-raw` for `parts-raw-except-authoring-comments`. Comment text in embedded templates stays subject to the ADR-0082 residue scan (invariant slugs only, no ADR citations or repo-identity literals) and the plain-punctuation gates. Doc paths are awf-given via the `.layout` namespace (never `.vars`); skills cite docs through `.layout.docs.*`/`.layout.workflowRef`, and a doc-gated skill without its doc is a refused config state at open (ADR-0081; formerly ADR-0013 render suppression). Catalog docs ship static default content with a per-doc section taxonomy; domain docs inject a generated index as forced body, and the glossary renders its table from its own sidecar data (ADR-0089, below). Toggleable docs and always-on singletons are one unified `Docs` collection (ADR-0061): each `DocEntry` carries its nature: `Mandatory` (always-on vs config-toggleable), `Path`/`TemplateKey`/`TID`, `DocumentMap`, and the `AgentsDoc` root-output flag. `.awf/` config only modulates that default (enable/disable, `local: true` suppression). Skill bodies likewise vary by declared capability rather than being byte-identical: the closed `targetSubagentTools` projection is consumed by seven skill templates, and only Pi declares `CapabilitySubagentTools`, so its review-dispatch prose differs from the other five targets'. A project enables adapters via the `targets` array (default `[claude]`), resolved against a registry of adapters (`claude`, `codex`, `copilot`, `cursor`, `gemini`, `pi`), each rendering skills and agents into its native layout with an optional bridge file (`claude`/`gemini`) and, for Pi, descriptor-owned `Outputs`; the path-keyed lock carries one entry per target with no hash collision, and Sync prunes a dropped target's whole directory tree.
