## Current state

The render engine is a marker-section overlay (`<!-- awf:section -->`) layered onto Go `text/template` with `missingkey=zero`, guarded by a hard publication-safety check that rejects any unresolved-variable placeholder. Convention parts are raw input (ADR-0034): only awf-owned template defaults are templated — each part slot is replaced by a brace-free sentinel before execution and the raw body restored verbatim afterwards, so a part renders byte-for-byte (a literal `{{` is never parsed) and is never variable-interpolated. Doc paths are awf-given via the `.layout` namespace (never `.vars`); skills cite docs through `.layout.docs.*`/`.layout.workflowRef`, and a doc-gated skill without its doc is a refused config state at open (ADR-0081; formerly ADR-0013 render suppression). Catalog docs ship static default content with a per-doc section taxonomy; domain docs are the one data-driven exception, injecting a generated index as forced body. Toggleable docs and always-on singletons are one unified `Docs` collection (ADR-0061): each `DocEntry` carries its nature — `Mandatory` (always-on vs config-toggleable), `Path`/`TemplateKey`/`TID`, `DocumentMap`, and the `AgentsDoc` root-output flag — and `.awf/` config only modulates that default (enable/disable, `local: true` suppression). The mandatory entries render the agent guide, the two ADR-system files (`README.md`, `template.md`), a plan-authoring guide (`plans/README.md`), the workflow / documentation-standard / agent-guide-authoring-standard docs (promoted from toggleable by ADR-0043), and (ADR-0059) a working-with-awf usage guide, each suppressible with a `local: true` sidecar. Every projection — `SingletonKinds()`, the `plainSingletons` render/validate set driving the existing `renderKind` primitive, the `.layout` singleton paths, and the toggleable-doc pool — derives from that one collection rather than a hand-maintained list, so adding a mandatory doc is a single `DocEntry` (ADR-0021, ADR-0020, ADR-0043, ADR-0061). An optional `.awf/bootstrap.sh` neutral config-tree singleton renders once when enabled (ADR-0040, relocated off the repo root by ADR-0047), pinned to the rendering binary's `project.Version`, excluded from the dead-reference scan, and pointing its unsupported-OS/arch failures at the manual-install README section. The `hooks` singleton (ADR-0048) likewise renders three inert git-hook payload scripts under `.awf/hooks/` as a unit when enabled — var-driven with graceful fallbacks and a pin-aware bootstrap shim, no overridable sections, excluded from the managed-markdown scans, and labeled per script in unset-var advisories; awf never activates them. `awf check` also scans every managed rendered markdown file for inline links whose file-relative target is missing on disk and fails on a `dead-reference` drift; the link extractor is the pure `internal/refs` package (ADR-0020). Per-kind dispatch across the render and check paths (enable array, catalog pool, declared sections, output path, labels) resolves through one ordered kind-descriptor table whose facets are accessor functions, absorbing the catalog map-vs-slice and adapter-vs-neutral path asymmetries in a single place (ADR-0027). `RenderAll` splits its passes (ADR-0037): neutral artifacts (docs, `AGENTS.md`, domain docs, the ADR-system singletons) render once, while adapter artifacts (skills, agents, and an optional `@AGENTS.md` bridge) render once per enabled adapter `Target`, the same body and frontmatter written to each runtime's path. A project enables adapters via the `targets` array (default `[claude]`), resolved against a registry holding `claudeTarget` and a bridge-less `cursorTarget` (Cursor reads `AGENTS.md` natively); the path-keyed lock carries one entry per target with no hash collision, and Sync prunes a dropped target's whole directory tree. Catalog specs carry per-artifact default `data:` merged under sidecars at sidecar resolution (present key wins, even null — ADR-0045), every var/data interpolation degrades to coherent generic prose when unset (the extended publication-safety contract), and the render context exposes the effective skills set as `.skills`; project-local skills and agents (ADR-0068) ride the same pipeline — an enabled name outside the standard pool that carries a declaring sidecar is synthesized into a per-project clone of `catalog.Standard` (`effectiveCatalog`; the package global is never mutated) as a `Base: true` entry with a single `content` section rendered from an awf-owned base template, so local artifacts get frontmatter, drift tracking, and part overrides for free, a standard entry is never shadowed, and an undeclared non-standard name still fails validation as a typo. ADR-0070 layers an unauthored-content signal onto the section model: a `stub` marker attribute flags must-replace defaults (distinct `— stub;` edit pointer), a whole-line `<!-- awf:stub -->` marker declares a convention part unauthored without breaking ADR-0034's verbatim contract, `StubSections`/`stubNotes` surface both as a non-failing, path-keyed advisory from `awf check`/`awf init` (one render pass with the unset-var notes, domain docs included), and a comment-anchored residual-marker guard makes any marker-shaped `awf:section`/`awf:end` token surviving assembly into the skeleton a hard render error instead of a silent leak — skeleton-scoped by design, since parts and data legitimately quote the form inline; the part-side sibling is ADR-0083's advisory: a trimmed part line *beginning with* a marker-shaped opener (scanned on the raw on-disk bytes, pre placeholder substitution, fenced code excluded via `refs.WithoutFences`) sets `SectionPlan.PartMarker` in `planSections`, rides `RenderedFile.markerParts` (hand-copied on the domain-doc path), and surfaces as a part-path-keyed, deduplicated `awf check`/`awf init` note whose text names fencing as the remedy — never a failure; `awf check` prints non-failing unset-var notes and fails on a `dead-skill-reference` drift — a rendered `<prefix>-<name>` token naming a known skill outside the effective set (ADR-0046). The render context also exposes `.commitScopes` — the display-formatted `audit.allowedScopes` list, empty when accept-any — folded into the config hash of referencing artifacts like `.skills`, so an `audit.allowedScopes` edit reflags exactly the templates that quote it (ADR-0051). A general `awf:include` directive (ADR-0052) splices awf-owned embedded partials under `templates/partials/` into any template before section parsing — the spliced text is thereafter indistinguishable from inline content, and `TemplateHash` is computed over the post-expansion source so a partial edit flags every including artifact stale; these partials are awf-owned templated content spliced pre-parse — on the templated side of ADR-0034's raw-vs-templated boundary, not project-supplied raw-convention input — so they carry no project config surface, and a missing, nested, or section-bearing partial is a hard render error. Its first use deduplicates the review-discipline spine shared by the three reviewer agents. A separate, narrower channel lets *raw* convention parts consume config-derived values without reopening the ADR-0034 brace trap: a closed `awf:`-namespaced sandbox placeholder set (ADR-0057) is literal-substituted in `planSections` (project layer, pre-`Assemble`) from a dynamic registry whose keys — `commitScopeList`, `commitScopeTable` (a `name | meaning` table over `audit.allowedScopes`, ADR-0056), `commitScopeSentence`, `prefix`, gate commands — are available only when non-empty; an unknown-or-empty key or a malformed near-miss (a `\{\{=\s*awf` residual guard) is a hard render error, and `render.ReferencesScopePlaceholder` folds resolved scope data into the config hash of any part using a scope placeholder so an `audit.allowedScopes` edit reflags it (`part-scopes-in-confighash`). A backslash escapes the opener via a NUL-sentinel pre-pass in `substitutePlaceholders` so a part can document the syntax literally, and `placeholderRegistry` hard-errors if any registry value itself carries the token (ADR-0058). One sandbox key is section-scoped rather than config-derived: `\{{=awf:sectionDefault}}` (ADR-0072) lets a convention part *extend* its section's template default instead of discarding it — `planSections` substitutes the brace-free `render.SectionDefaultSentinel`, `Assemble` splits the part body at each occurrence and splices the section's raw default source between the verbatim fragments so Execute templates the default in place, and re-injecting a stub-attributed section's default is a hard render error (`CheckSectionDefaultStubs`), since a stub default is an authoring prompt, not shippable prose. A third neutral config-tree unit, `.awf/memory/.gitignore` (ADR-0069), renders unconditionally — no config gate, no catalog spec — as a self-ignoring gitignore backing the working-memory convention; `injectBanner` stamps it with a `#`-comment banner via a branch keyed on its template id, and that id is excluded from the managed-markdown scans like the bootstrap and hook payloads. The catalog also declares each artifact's unconditional chain-skill coupling (`requiresSkills`, ADR-0080); a catalog-derived test sweep renders every template under empty data and holds those declarations exact. Enabled now means rendered: the ADR-0013 doc-gate render suppression is gone — a doc-gated skill without its doc is a refused config state, not a silently-unrendered one (ADR-0081), and the render context's effective skills set is exactly the enabled set.
