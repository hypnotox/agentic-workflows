## Current state

The topic producer renders each valid pair to `<docsDir>/topics/<domain>/<topic>.md`, emits a title-and-summary-sorted `<docsDir>/topics/<domain>/index.md`, and adds compact topic navigation to the owning domain page in place of a per-domain ADR index. Authored topic Markdown uses the raw-part convention and authoring-comment strip rather than YAML interpolation. Topic pages and indexes are ordinary managed Markdown output-plan nodes, so manifest hashing, brownfield backup, drift, and prune share the existing lifecycle with no topic-specific lock field. Together with the generated decision index `INDEX.md`, they are the active authority surface that replaced the retired `ACTIVE.md` and per-domain ADR indexes; the one-time project-atomic cutover deleted those legacy outputs through the recoverable upgrade transaction.

ADR-0132 adds the core exploring skill with six-target semantic rendering. Pi's capability-selected branch invokes the structured `subagent_explore` contract; all non-Pi targets carry the same breadth, detail, boundary, outcome, and reporting semantics through generic native delegation.

ADR-0141 gives every Pi subagent role strict optional per-call model routing, bounds independent exploration at ten active FIFO-scheduled children with abort-aware queueing, and blocks reconstructable mixed implementation batches at the current-leaf tool-call preflight seam.

ADR-0127 changes what the `reviewing-impl` skill renders: it now passes the session range it already derives (`${baseSha}..${headSha}`) to `awf audit` instead of auditing "the branch", so the terminal review audits the session it just reviewed under any branching strategy.

The render engine is a marker-section overlay (`<!-- awf:section -->`) layered onto Go `text/template` with `missingkey=zero`, guarded by a hard publication-safety check that rejects any unresolved-variable placeholder. Convention parts are raw input (ADR-0034): only awf-owned template defaults are templated: each part slot is replaced by a brace-free sentinel before execution and the raw body restored verbatim afterwards, so a part renders byte-for-byte (a literal `{{` is never parsed) and is never variable-interpolated. ADR-0121 adds the one removal to the substitutions: whole-line `awf:comment` authoring comments are stripped from part bodies (and template sources) at ingestion, retiring `parts-raw` for `parts-raw-except-authoring-comments`. Comment text in embedded templates stays subject to the ADR-0082 residue scan (invariant slugs only, no ADR citations or repo-identity literals) and the plain-punctuation gates. Doc paths are awf-given via the `.layout` namespace (never `.vars`); skills cite docs through `.layout.docs.*`/`.layout.workflowRef`, and a doc-gated skill without its doc is a refused config state at open (ADR-0081; formerly ADR-0013 render suppression). Catalog docs ship static default content with a per-doc section taxonomy; domain docs inject a generated index as forced body, and the glossary renders its table from its own sidecar data (ADR-0089, below). Toggleable docs and always-on singletons are one unified `Docs` collection (ADR-0061): each `DocEntry` carries its nature: `Mandatory` (always-on vs config-toggleable), `Path`/`TemplateKey`/`TID`, `DocumentMap`, and the `AgentsDoc` root-output flag. `.awf/` config only modulates that default (enable/disable, `local: true` suppression). The mandatory entries render the agent guide, the two ADR-system files (`README.md`, `template.md`), a plan-authoring guide (`plans/README.md`), the workflow / documentation-standard / agent-guide-authoring-standard docs (promoted from toggleable by ADR-0043), and (ADR-0059) a working-with-awf usage guide, each suppressible with a `local: true` sidecar. Every projection (`SingletonKinds()`, the `plainSingletons` render/validate set driving the existing `renderKind` primitive, the `.layout` singleton paths, and the toggleable-doc pool) derives from that one collection rather than a hand-maintained list, so adding a mandatory doc is a single `DocEntry` (ADR-0021, ADR-0020, ADR-0043, ADR-0061). An optional `.awf/bootstrap.sh` neutral config-tree singleton renders once when enabled (ADR-0040, relocated off the repo root by ADR-0047), pinned to the rendering binary's `project.Version`, excluded from the dead-reference scan, and pointing its unsupported-OS/arch failures at the manual-install README section. The `hooks` singleton (ADR-0048) likewise renders three inert git-hook payload scripts under `.awf/hooks/` as a unit when enabled: var-driven with graceful fallbacks and a pin-aware bootstrap shim, no overridable sections, excluded from the managed-markdown scans, and labeled per script in unset-var advisories; awf never activates them. `awf check` also scans every managed rendered markdown file for inline links whose file-relative target is missing on disk and fails on a `dead-reference` drift; the link extractor is the pure `internal/refs` package (ADR-0020). Per-kind dispatch across the render and check paths (enable array, catalog pool, declared sections, output path, labels) resolves through one ordered kind-descriptor table whose facets are accessor functions, absorbing the catalog map-vs-slice and adapter-vs-neutral path asymmetries in a single place (ADR-0027). `RenderAll` splits its passes (ADR-0037): neutral artifacts (docs, `AGENTS.md`, domain docs, the ADR-system singletons) render once, while adapter artifacts (skills, agents, and an optional `@AGENTS.md` bridge) render once per enabled adapter `Target`, then encoded into that runtime's dialect (ADR-0122): an agent is a format-neutral model (a literal `name`, a separately rendered `description`, a section-rendered instruction body) that `encodeMarkdownAgent` writes as YAML-frontmatter Markdown for `claude`/`copilot`/`cursor`/`gemini`/`pi` and `encodeTOMLAgent` writes as a typed `name`/`description`/`developer_instructions` profile for `codex`, which carries no frontmatter at all; the TOML encoder never parses rendered Markdown (pinned by `TestTOMLEncoderDoesNotDependOnMarkdownParser`). Skill bodies likewise vary by declared capability rather than being byte-identical: the closed `targetSubagentTools` projection is consumed by seven skill templates, and only Pi declares `CapabilitySubagentTools`, so its review-dispatch prose differs from the other five targets'. A project enables adapters via the `targets` array (default `[claude]`), resolved against a registry of adapters (`claude`, `codex`, `copilot`, `cursor`, `gemini`, `pi`), each rendering skills and agents into its native layout with an optional bridge file (`claude`/`gemini`) and, for Pi, descriptor-owned `Outputs`; the path-keyed lock carries one entry per target with no hash collision, and Sync prunes a dropped target's whole directory tree. Catalog specs carry per-artifact default `data:` merged under sidecars at sidecar resolution (present key wins, even null, ADR-0045), every var/data interpolation degrades to coherent generic prose when unset (the extended publication-safety contract), and the render context exposes the effective skills set as `.skills`; project-local skills and agents (ADR-0068) ride the same pipeline: an enabled name outside the standard pool that carries a declaring sidecar is synthesized into a per-project clone of `catalog.Standard` (`effectiveCatalog`; the package global is never mutated) as a `Base: true` entry with a single `content` section rendered from an awf-owned base template, so local artifacts get frontmatter, drift tracking, and part overrides for free, a standard entry is never shadowed, and an undeclared non-standard name still fails validation as a typo. ADR-0070 layers an unauthored-content signal onto the section model: a `stub` marker attribute flags must-replace defaults (distinct `: stub;` edit pointer), a whole-line `<!-- awf:stub -->` marker declares a convention part unauthored without breaking ADR-0034's verbatim contract, `StubSections`/`stubNotes` surface both as a non-failing, path-keyed advisory from `awf check`/`awf init` (one render pass with the unset-var notes, domain docs included), and a comment-anchored residual-marker guard makes any marker-shaped `awf:section`/`awf:end` token surviving assembly into the skeleton a hard render error instead of a silent leak; skeleton-scoped by design, since parts and data legitimately quote the form inline; the part-side sibling is ADR-0083's advisory: a trimmed part line *beginning with* a marker-shaped opener (scanned on the raw on-disk bytes, pre placeholder substitution, fenced code excluded via `refs.WithoutFences`) sets `SectionPlan.PartMarker` in `planSections`, rides `RenderedFile.markerParts` (hand-copied on the domain-doc path), and surfaces as a part-path-keyed, deduplicated `awf check`/`awf init` note whose text names fencing as the remedy, never a failure; `awf check` prints non-failing unset-var notes (since ADR-0087 only for a var whose key is present with an empty or null value, an absent key being the acknowledged decline, with the note text naming both exits) and fails on a `dead-skill-reference` drift: a rendered `<prefix>-<name>` token naming a known skill outside the effective set (ADR-0046). The render context also exposes `.commitScopes` (the display-formatted `audit.allowedScopes` list, empty when accept-any) folded into the config hash of referencing artifacts like `.skills`, so an `audit.allowedScopes` edit reflags exactly the templates that quote it (ADR-0051). A general `awf:include` directive (ADR-0052) splices awf-owned embedded partials under `templates/partials/` into any template before section parsing; the spliced text is thereafter indistinguishable from inline content, and `TemplateHash` is computed over the post-expansion source so a partial edit flags every including artifact stale; these partials are awf-owned templated content spliced pre-parse (on the templated side of ADR-0034's raw-vs-templated boundary, not project-supplied raw-convention input) so they carry no project config surface, and a missing, nested, or section-bearing partial is a hard render error. Its first use deduplicates the review-discipline spine shared by the three reviewer agents. A separate, narrower channel lets *raw* convention parts consume config-derived values without reopening the ADR-0034 brace trap: a closed `awf:`-namespaced sandbox placeholder set (ADR-0057) is literal-substituted in `planSections` (project layer, pre-`Assemble`) from a dynamic registry whose keys are available only when non-empty: `commitScopeList`, `commitScopeTable` (a `name | meaning` table over `audit.allowedScopes`, ADR-0056), `commitScopeSentence`, `prefix`, gate commands, and `gatedCommands` (the binary-version-gated command list generated from the `internal/clispec` table so it cannot drift, ADR-0094, also exposed as an `AGENTS.md` render key). An unknown-or-empty key or a malformed near-miss (a `\{\{=\s*awf` residual guard) is a hard render error, and `render.ReferencesScopePlaceholder` folds resolved scope data into the config hash of any part using a scope placeholder so an `audit.allowedScopes` edit reflags it (`part-scopes-in-confighash`). A backslash escapes the opener via a NUL-sentinel pre-pass in `substitutePlaceholders` so a part can document the syntax literally, and `placeholderRegistry` hard-errors if any registry value itself carries the token (ADR-0058). One sandbox key is section-scoped rather than config-derived: `\{{=awf:sectionDefault}}` (ADR-0072) lets a convention part *extend* its section's template default instead of discarding it: `planSections` substitutes the brace-free `render.SectionDefaultSentinel`, `Assemble` splits the part body at each occurrence and splices the section's raw default source between the verbatim fragments so Execute templates the default in place, and re-injecting a stub-attributed section's default is a hard render error (`CheckSectionDefaultStubs`), since a stub default is an authoring prompt, not shippable prose. A third neutral config-tree unit, `.awf/memory/.gitignore` (ADR-0069), renders unconditionally (no config gate, no catalog spec) as a self-ignoring gitignore backing the working-memory convention; `injectBanner` stamps it with a `#`-comment banner via a branch keyed on its template id, and that id is excluded from the managed-markdown scans like the bootstrap and hook payloads. The catalog also declares each artifact's unconditional chain-skill coupling (`requiresSkills`, ADR-0080); a catalog-derived test sweep renders every template under empty data and holds those declarations exact. Enabled now means rendered: the ADR-0013 doc-gate render suppression is gone: a doc-gated skill without its doc is a refused config state, not a silently-unrendered one (ADR-0081), and the render context's effective skills set is exactly the enabled set. The render layer feeds the ADR-0086 consumption checks: `RenderedFile` carries its artifact identity and the `\{{=awf:gateCmd}}`/`\{{=awf:checkCmd}}` part-placeholder var refs (parts are sentinel-substituted raw bodies the assembled-source scan cannot see), the generated domain docs retain their assembled source for the var-consumption union, and `internal/render` exposes the `ReferencedDataKeys`/bare-reference extractors beside `ReferencedVars`. ADR-0088 adds a fourth doc nature: a `Generated` flag on `DocEntry` marks a Mandatory doc rendered outside `RenderAll` from computed project state, excluded from `plainSingletons` and hash checking, regeneration-checked like INDEX.md and the domain docs. Its first occupant is the always-on configuration reference (`docs/config-reference.md`): `generateConfigReference` renders the embedded template with four injected data collections built from `internal/configspec` (the compile-time adopter-facing description authority: config keys, sidecar fields, per-artifact data keys, var entries derived verbatim from the catalog descriptors with configspec-owned availability clauses) plus the live consumption graph (enabled consumers from the ADR-0086 union; dormant hints from a raw-template scan over the full catalog, guarded by a no-`.vars`-in-partials test). The template consumes only dedicated `.data` collections, never bare `.vars`/`.data` (`config-reference-no-bare-vars`), so it cannot widen the consumption checks' conservative escapes; its sidecar is sections/local-only (`data:` refuses at open, `config-reference-data-rejected`), its `intro` section is the one part-overridable slot, and the generated file joins the dead-reference/dead-skill scans, the stub/marker advisories, and `isGeneratedIndex`'s first-takeover note. ADR-0089 adds the *sidecar-derived* computed-doc seam, deliberately distinct from that regeneration model: an optional `transform` on `renderKindSpec` mutates the resolved sidecar after the defaults merge, upstream of both `renderTarget` and `artifactConfigHash` (the ADR-0045 both-consumers pattern), so a change to the computation itself reflags the doc while it stays in the ordinary `RenderAll`/lock model. Its first occupant is the glossary: `docDataTransform` replaces the authored `data.terms` map (both yaml.v3 map shapes accepted) with always-sorted (case-insensitive), pipe-escaped table rows rendered from plain non-section template text between the empty-default `prepend`/`append` framing sections; no part can override the table. Content violations (empty or newline-carrying terms/meanings, non-string values or keys, case-insensitive duplicate terms) are hard render errors naming `.awf/docs/glossary.yaml` and the offending key; absent, null, or empty terms degrade to a placeholder line naming the authoring surface (`glossary-terms-sorted`, `glossary-terms-validated`, `glossary-table-forced`). ADR-0099 makes the pitfalls doc the transform's second occupant: `pitfallsTransform` replaces the authored `data.pitfalls` entry list (`{title, domains, related, body}` mappings) with `## `-section markdown in authored order (a YAML sequence is already deterministic, so no sort), each entry carrying an optional italic `Domains` line and a `Related` line of plain `ADR-NNNN` references, rendered between the same empty-default `prepend`/`append` framing; a non-list shape, an empty or newline-bearing title, or an empty body is a hard render error naming `.awf/docs/pitfalls.yaml` (`pitfall-data-validated`).

ADR-0124 replaces the distributed render/lifecycle output discovery with a deterministic internal output plan. Each path is a writing node or local reservation with an explicit output policy. Render, sync, lock manifests, prune protection, planned-output reporting, and drift checks consume that node set; Markdown, TOML, TypeScript, generated, and local behavior is selected by policy rather than path suffix or template ID. The configuration reference depends on preceding ordinary/domain nodes without self-dependence. Target descriptors validate closed capabilities, bridge pairs, and safe output declarations before planning; equivalent recipes coalesce while differing recipes fail before a write.

The rendered-output quality bar has a deterministic review surface (ADR-0090): the committed full-surface example adopter `examples/sundial/` re-renders on every `./x sync`, so a template change lands as a reviewable rendered diff over a realistic adoption in the same commit, and a schema bump must run `awf upgrade` there before `./x check` goes green, an in-repo migration rehearsal ahead of any external adopter.

ADR-0091 extends the ADR-0068 local-artifact model to a third kind. `effectiveCatalog` now clones the `Docs` map too (`invariant: local-doc-catalog-clone`), and an enabled `docs:` name outside the standard pool that carries a declaring sidecar is synthesized into a `DocEntry` with a single freeform `content` section, resolved through a new `p.docTID` (reading the effective catalog, not the package global) to an awf-owned base doc template `templates/docs/_base.md.tmpl`. Synthesis lifts the sidecar's `title`/`description` into the entry (with a name-derived title and generic description default) so a local doc always carries a non-empty document-map label (`invariant: local-doc-map-fields`) and inherits the ADR-0020 dead-link scan like any catalog doc; the docs render pass gains a `defaults` injection so the base template's title has a fallback. Doc names are path-aware (`guides/ci`), validated by `config.ValidateDocName` in the config domain. `RenderAll` now fails loudly (`assertNoDuplicateOutputPaths`) when two artifacts resolve to the same output path, so a path-aware doc name landing on awf's reserved `decisions/`/`plans/`/`domains/` territory is caught rather than silently overwriting. The `releasing` toggleable catalog doc rides the ordinary catalog path: a single stub-classified `content` section (ADR-0070), no imposed structure; and awf dogfoods it, its `docs/releasing.md` runbook now generated output listed in the document map via `resolvedDocs` rather than the retired `docMap` sidecar escape hatch.

ADR-0100 adds **in-place-editable sections**, a second adopter override channel beside convention parts: a section declared with the `inplace` marker attribute has its body read back from the *existing rendered output* (bounded by its `awf:edit-in-place` provenance pointer and awf's next registered section pointer, matched by that pointer's expected string, never a generic pointer shape) and preserved across syncs, while awf regenerates every other section and the file structure from the template; the file is drift-checked by regeneration-with-read-back rather than the frozen `OutputHash`, made a first-class `RegenChecked` attribute on `RenderedFile`/`manifest.Entry` that also absorbed the former hardcoded generated-index path list. Two shell-script properties are rendered per target off the one `#!`-shebang predicate (shared with `injectBanner`): the surviving `awf:edit`-family pointers take the target's comment syntax: a `#`-line comment for a shebang script, HTML otherwise (`render.CommentStyle`), so an in-place shell file is valid, and a rendered `#!` file is written mode `0755` (executable, enforced every sync), so a runner is runnable as `./x`.
