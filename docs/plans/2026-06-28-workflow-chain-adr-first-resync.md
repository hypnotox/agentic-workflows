# Plan: Workflow chain → ADR-first ordering + visible resync + lifecycle reconciliation

Implements **ADR-0028** (`docs/decisions/0028-workflow-chain-adr-first-visible-resync.md`, status
`Proposed`). The design and rationale live in the ADR — this plan is the execution record only.

## Goal

Make the rendered workflow chain ADR-first, surface the plan↔ADR resync as a first-class step,
record the many-to-one ADR↔plan relationship and the planning→ADR back-edge, and reconcile the
ADR/plan lifecycle state rules — all by editing awf's shipped templates and re-rendering. ADR-0028
flips `Proposed → Implemented` in the final commit (Decision 6: the flip is owned by the
implementation step's final commit).

## Architecture summary

All substance lives in `templates/` (the shipped defaults) plus two hand-maintained files
(`README.md`, `.awf/docs/parts/glossary/terms.md`) and one test. Rendered outputs (`AGENTS.md`,
`docs/workflow.md`, `.claude/skills/*`, `docs/glossary.md`, `docs/domains/adr-system.md`,
`docs/decisions/ACTIVE.md`) are regenerated by `./x sync` and committed alongside their sources —
never hand-edited. No Go production code changes; the only `*.go` edit is the golden test plus two
`// invariant:` marker comments backing the ADR's tagged invariants.

## Tech stack

- Go 1.26 (module `github.com/hypnotox/agentic-workflows`); only `internal/project/spine_test.go` is touched.
- awf template tree under `templates/`; render via `./x sync`; drift + invariant gate via `./x check`; full gate `./x gate`.
