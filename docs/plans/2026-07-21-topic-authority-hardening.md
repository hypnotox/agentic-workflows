# Plan: Topic Authority Hardening

## Goal

Close every confirmed correctness and integration defect from the topic-authority audit and implement
[ADR-0139](../decisions/0139-explicit-current-state-authority-provenance.md), with regression-backed
fresh and brownfield adoption, immutable ADR history, complete queries/context, explicit agent gates,
and current documentation. Non-goals are changing marker scan-universe semantics or redesigning
cross-schema historical range audit.

## Architecture summary

Add one manifest-owned lock-state classifier and an init-only authority seed, then make project,
upgrade, snapshot, and staged consumers use it. Tighten lifecycle validation through ADR-owned
semantic comparison helpers. Reuse the cutoff-aware current-state snapshot for historical queries,
project canonical topic fields into context, and update agent templates to run staged validation
explicitly. Each behavior lands with its regression and active documentation; ADR-0139's four claims,
proof markers, generated outputs, and terminal status land together in the final transaction.
