# Plan: Trim plan header taxonomy to three fields

Implements ADR-0108: drop the `Tech stack` header field and the standalone `positioning` section
from the plan-artifact taxonomy, fold `positioning`'s one-liner plus a one-line non-goals statement
into `Goal`, and interpolate the configured `gateCmd` in the plans template. The design and rationale
live in ADR-0108; this plan is the execution record. Non-goals: no change to phases, task granularity,
the Verification/Notes tails, or the plan lifecycle; no auto-migration of adopter part overrides.

## Goal

Bring every source surface of the plan taxonomy into line with ADR-0108's three-field header, update
the one backed invariant test, and re-render all targets, in a single coupled commit that also flips
ADR-0108 and this plan to `Implemented`.

## Architecture summary

Eight hand-edited source surfaces (one template rewrite, one catalog `Sections` list, one golden
test, four prose surfaces, one changelog entry), then `./x sync` regenerates every rendered
output: the root and `examples/sundial` copies of `docs/plans/template.md` and `docs/plans/README.md`,
the `.claude`/`.cursor` skill and agent renders, `AGENTS.md`, `docs/decisions/ACTIVE.md`, and both
`awf.lock` files. The template rewrite, the catalog list, and the golden test are mutually gate-coupled
(the `plans-template-taxonomy` golden test and the `adr-singleton-section-parity` test each fail unless
template markers, the catalog `Sections` list, and the asserted heading set all move together), and the
single atomic `awf.lock` forces every rendered-output change into one sync; so this lands as one
coupled commit, not sliced phases.

## Notes

**Findings at freeze:** One under-enumeration surfaced during execution. The plan's task list
(and ADR-0108's fan-out) missed two tests coupled to `gateCmd`'s *consumer set*, which changed
because the plans-template now references `.vars.gateCmd`: `internal/project/configreference_test.go`
(golden "Consumed by" list for `gateCmd` gained `plans-template`) and
`internal/project/unused_test.go` `TestPartPlaceholderConsumesVar` (its premise that `gateCmd` is
otherwise-unused in the fixture broke, since the always-on `plans-template` singleton now consumes it;
switched the isolation var to `checkCmd`). Both are legitimate premise updates, folded into the
same coupled commit. The changelog approach was corrected earlier, during the plan↔ADR resync, from a
new Breaking-changes entry to an amendment of the still-unreleased plans-template entry (Task 1.8).

Out-of-scope items, follow-ups, and findings surfaced during implementation are recorded here at the
`status: Implemented` flip (Task 1.10). The example adopter's
committed plan (`examples/sundial/docs/plans/2026-07-07-decimal-degrees-cli.md`) keeps its inline
`**Tech stack:**` field intentionally: ADR-0108 grandfathers existing plans, `awf check` never
validates plan bodies, and the example stays zero-notes.
