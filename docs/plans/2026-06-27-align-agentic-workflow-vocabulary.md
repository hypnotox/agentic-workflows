# Plan: Align agentic-workflow vocabulary across instruction surfaces

## Goal

Apply field-correct terminology and structure across both the shipped templates (the
rendered standard every adopter gets) and awf's own positioning prose. This is a pure
wording/structure precision pass — the standard's **rules do not change**, only how
precisely they are named. Non-load-bearing (no ADR); see the brainstorm for the five gaps.

Guiding principle, applied to every edit: **replace an imprecise phrase with the precise
term; never append explanatory or dated clauses** (no "the field has converged", no "as
of 2026"). Keep net length flat or shrinking.

## The five gaps (from the brainstorm)

1. **`panel` is inconsistent** — some skills positively call the reviewer "the lens panel"
   while others negate "a panel". Standardise on: a **single lens-diverse review subagent**,
   never a panel.
2. **Reviewer independence is unnamed** — name the reviewer as an **independent,
   fresh-context** review distinct from the implementer/generator.
3. **Orchestrator framing is loose** — name the main session as the **orchestrator** holding
   coordination context while isolated subagents do scoped work.
4. **The gate is described functionally, never named** — name it the **deterministic gate**
   (the deterministic checks wrapping the probabilistic agent).
5. **awf Identity paragraph** — sweep to carry the same vocabulary.

## Architecture summary

No production code changes. Phase 2 also updates two test assertions
(`internal/project/spine_test.go`, `internal/project/golden_test.go`) that hard-assert the
code-reviewer description string being renamed — they must change in the same commit to keep
the gate green. All other edits are to embedded template prose (`templates/`) plus one awf-own
convention part (`.awf/parts/agents-doc/identity.md`), then a re-render. The render loop is
unchanged. Each edit is a within-section prose change — no sections added/removed, so no
section-taxonomy parity or publication-safety guard is touched (verified in the
grounding-check). After each phase: `./x sync` re-renders, `./x check` confirms no drift,
and the source + regenerated files + `.awf/awf.lock` are committed together.

## Tech stack

- Go 1.26 (tool only; production code not touched).
- `text/template` rendering via `awf sync` (`./x sync`), drift via `awf check` (`./x check`).
- Changed artifacts are markdown (skills, agents, docs, AGENTS.md) plus, in Phase 2, two test
  files (`internal/project/spine_test.go`, `internal/project/golden_test.go`). **The gate is
  not skipped on docs-only commits** — the pre-commit hook runs `./x check` (drift) then
  `./x gate` (full `go test ./...` + 100% coverage + vet + lint) on **every** commit; both
  must pass. Phases 1 and 3 pass the gate because no test asserts their edited prose; Phase 2
  passes only after the two assertions are updated in the same commit.
