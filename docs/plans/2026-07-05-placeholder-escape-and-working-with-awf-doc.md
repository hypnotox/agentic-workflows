## Phase 1 — ADR-0058 escape + value-guard

- [ ] **1.1 Escape pre/post pass in `substitutePlaceholders`.** In `internal/project/placeholders.go`, add a package var and rewrite the body so an escaped opener is hidden before the two passes and restored after:
  ```go
  const awfEscSentinel = "\x00awf-esc\x00"

  var awfEscapeRE = regexp.MustCompile(`\\(\{\{=\s*awf)`)
  ```
  In `substitutePlaceholders`, immediately after the fast-path guard `if !strings.Contains(body, "{{=") { return body, nil }`, protect escaped openers, then (unchanged) run the strict `ReplaceAllStringFunc` and the residual check on `out`, then restore:
  ```go
  // Protect \{{=…awf escapes: consume the backslash, hide the {{= from both
  // passes, keep the \s*awf tail in-body. invariant: escaped-placeholder-literal
  body = awfEscapeRE.ReplaceAllString(body, awfEscSentinel+"$1"[2:]) // see note
  ```
  Note: `$1` captures `{{=\s*awf`; we want to drop the backslash and hide only `{{=`. Implement with a func replace for clarity instead of the expression above:
  ```go
  body = awfEscapeRE.ReplaceAllStringFunc(body, func(m string) string {
      // m starts with '\' then '{{=' then optional ws + 'awf'. Drop the
      // backslash; stand the '{{=' behind the sentinel; keep the rest.
      rest := m[1:]                    // '{{=' + ws + 'awf'
      return awfEscSentinel + rest[3:] // sentinel + (ws + 'awf'); '{{=' hidden
  })
  ```
  After the residual-guard block, before `return out, nil`, restore: `out = strings.ReplaceAll(out, awfEscSentinel, "{{=")`. The sentinel is created and fully restored here, never reaching `Assemble`.

  ```
  <!-- awf:section overview -->
  # Working with awf

  This project's skills, agents, docs, and `AGENTS.md` are generated by awf from the committed
  `.awf/` config tree. Every rendered file carries a `GENERATED by awf` banner — never hand-edit one;
  change `.awf/` and re-render. This guide covers working with an already-adopted project; adopting
  awf into a new repo is documented in the awf project itself.
  <!-- awf:end -->

  <!-- awf:section commands -->
  ## Commands

  - `awf sync` — re-render every artifact from `.awf/` and update `.awf/awf.lock`.
  - `awf check` — fail if any rendered file has drifted from the config (also gates the schema version).
  - `awf add <kind> <name>` / `awf remove <kind> <name>` — toggle a skill, agent, doc, domain, target, bootstrap, or hooks artifact.
  - `awf list` — show the catalog and what is enabled.
  - `awf new adr "<title>"` — scaffold the next ADR.
  - `awf upgrade` — migrate the config tree after upgrading the awf binary.
  - `awf audit` — report Conventional-Commits / workflow-conformance findings over the branch (advisory).
  - `awf commit-gate <file>` — validate one commit message (used by a commit-msg hook).
  - `awf changelog` — query the changelog by version or range.
  <!-- awf:end -->

  <!-- awf:section config-and-overrides -->
  ## Config and overrides

  Configuration lives in `.awf/`: a `config.yaml` skeleton (prefix, vars, enable arrays, targets,
  audit, invariants), per-artifact sidecar YAMLs, and convention parts. To change one section of a
  rendered artifact, drop a **convention part** at `.awf/<kind>/parts/<target>/<section>.md` (for a
  doc, `.awf/docs/parts/<name>/<section>.md`; for an always-on singleton,
  `.awf/parts/<kind>/<section>.md`). A part **replaces that section's body** and inherits the rest of
  the template default. Convention parts are raw input: their content renders verbatim and is never
  run through the template engine — see [{{ .layout.docStandard }}]({{ .layout.docStandard }}) for the
  authoring rules.
  <!-- awf:end -->

  <!-- awf:section placeholders -->
  ## Placeholders in overrides

  A convention part may splice in awf-derived values through the `awf:`-namespaced placeholder
  {{ "{{=awf:KEY}}" }}. It is a literal substitution (not the template engine), so a stray `{{` in
  prose stays inert. A key is available only when its value is **non-empty** this render; an unknown
  or empty key, or a malformed near-miss, is a hard error that names the available keys. The keys:

  | key | renders |
  |---|---|
  | `commitScopeList` | the allowed commit-scope names, comma-separated |
  | `commitScopeTable` | a markdown table of scope names and meanings |
  | `commitScopeSentence` | a one-sentence statement of the allowed scopes |
  | `prefix` | the project's artifact prefix |
  | `gateCmd` | the configured pre-commit gate command |
  | `checkCmd` | the configured drift-check command |

  To write the placeholder syntax literally in a part (for documentation), put a backslash before it:
  {{ "\\{{=awf:commitScopeTable}}" }} renders as the literal {{ "{{=awf:commitScopeTable}}" }}.
  <!-- awf:end -->

  <!-- awf:section sync-and-drift -->
  ## Keeping in sync

  After any edit to `.awf/`, run `awf sync` to re-render, then `awf check` to confirm no drift, and
  commit the rendered files alongside the config change. `awf check` compares each rendered file's
  template and config against `.awf/awf.lock`; a mismatch is `stale` drift. Wire `awf check` and your
  gate into CI so drift and rule violations block a merge.
  <!-- awf:end -->
  ```
