# Plan: doc-standard conformance sweep

Brings awf's pre-existing managed docs into conformance with `docs/doc-standard.md` (ADR-0018) —
the deferred sweep noted in ADR-0018's scope boundary. Non-ADR work: the rules being applied are
already decided (ADR-0018); this plan applies them across existing sites.

## Goal

Stop re-narrating tooling mechanisms that an ADR owns (chiefly `missingkey=zero` / publication
safety, ADR-0001; the schema-version gate, ADR-0010) and trim editorializing/restatement, per
`doc-standard`'s rules. Fix the violations an audit found in the architecture doc (this repo's
convention parts, local) and the shipped `workflow`/`testing` template defaults.

## Architecture summary

Docs-only, no code. Two blast radii, one per phase: Phase 1 edits `.awf/docs/parts/architecture/*`
(renders only this repo's `docs/architecture.md`); Phase 2 edits `templates/docs/{workflow,testing}.md.tmpl`
(the shipped standard — changes every adopter's rendered `workflow`/`testing`). All edits are within
existing `awf:section` bodies (no section add/remove → `docs_sections_test.go` parity untouched); no
rewrite introduces the literal `<no value>` token (publication-safety guard). `development.md.tmpl`
and the skeleton-prompt sections need no change. Grounding-check confirmed every old-string verbatim
and every added ADR citation (0001 = engine/missingkey/publication-safety; 0010 = schema gate;
0015 = `awf:edit`; 0016 = Target seam).

## Tech stack

- No Go changes. `awf sync`/`check` via `./x sync` / `./x check`. The pre-commit hook runs `./x check`
  then `./x gate` on every commit; both must pass (the gate is trivially green — no code changed).
