# Plan: Catalog → Go + unified doc model (ADR-0060 + ADR-0061)

**ADRs:** [ADR-0060](../decisions/0060-catalog-representation-moves-from-embedded-yaml-to-compile-time-go.md) (catalog representation moves embedded YAML → compile-time Go), [ADR-0061](../decisions/0061-unified-catalog-doc-model-for-toggleable-docs-and-singletons.md) (merge toggleable docs + always-on singletons into one `DocEntry` collection). Rationale lives in the ADRs; this plan is execution only.

## Goal

Phase 1 (ADR-0060): move the whole embedded `templates/catalog.yaml` into a compile-time Go value `catalog.Standard`, delete `catalog.Load` and the YAML, make `internal/catalog` an fs-free leaf — behaviour-preserving (byte-identical rendered output; `.awf/awf.lock` stable). Phase 2 (ADR-0061): merge `catalog.Docs` and `catalog.Singletons` into one `Docs map[string]DocEntry` carrying `Mandatory`/`Path`/`TemplateKey`/`TID`/`DocumentMap`/`AgentsDoc`; make every projection (`SingletonKinds`, `plainSingletons`, `Layout` paths + `templateMap` keys, the toggleable-doc pool, the two formerly-hardcoded tests, the `render.go` comment) derive from it, so adding a mandatory doc is one `DocEntry`.

## Architecture summary

- **P1** replaces the runtime `yaml.Unmarshal(catalog.yaml)` with a Go literal `var Standard = &Catalog{…}` in `internal/catalog/standard.go`; every `catalog.Load(templates.FS)` caller reads `catalog.Standard`. `catalog.yaml` and its embed token are deleted.
- **P2** introduces `catalog.DocEntry`, folds `Singletons` into `Docs`, and turns `SingletonKinds`/`plainSingletons`/`Layout` singleton paths/the `docs` pool into derivations. The map-merge deletes `cat.Singletons`, whose keys back two ADR-0043 invariants; since `awf check` requires those backings until ADR-0061 is `Implemented`, the merge + invariant retirement + ADR-0061 status flip land in **one atomic commit**.

## Phase 1 — ADR-0060: catalog YAML → Go (behaviour-preserving)

- [ ] **1.1 Add `internal/catalog/standard.go` with the Go catalog value.** Create `var Standard = &Catalog{…}` transcribing **every** entry of the current `templates/catalog.yaml` 1:1 into Go literals, same struct shapes (`SkillSpec`/`TargetSpec`/`DocSpec`/`VarDescriptor`). Mapping rules, applied verbatim:
  - A YAML mapping `key: value` → a struct field. A YAML block scalar list → `[]string{…}`.
  - The freeform `data:` bag → `map[string]any{…}`, reproducing yaml.v3's runtime shapes so `ConfigHash` is unchanged: a YAML scalar string → Go `string`; a YAML sequence → `[]any{…}`; a nested mapping → `map[string]any{…}`. Do **not** use typed sub-structs or `[]string`/`map[string]string` for `data` values — those marshal to different bytes and would churn `.awf/awf.lock`.
