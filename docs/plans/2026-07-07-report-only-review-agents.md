# Plan: Report-only review agents

Implements [ADR-0074](../decisions/0074-report-only-review-agents.md). Design rationale lives
there; this plan is the execution record only.

## Goal

Make the three chain review subagents (`adr-reviewer`, `plan-reviewer`, `code-reviewer`)
**report-only judges**: they emit findings + a digest and never edit, commit, or re-review. The
dispatching `awf-reviewing-*` skills take over fix application (mechanical directly / reasoned with a
one-line rationale / user-decision escalated) and run exactly one fresh verify-pass dispatch. Retire
the now-dead `fixesAsCommits` catalog key and the agent-side 3-round soft cap.

## Architecture summary

The fix-application instructions are centralised in the ADR-0052 shared spine
(`templates/partials/review-spine-{head,tail}.md`, spliced into all three reviewer agent templates
via `awf:include`) plus each reviewing skill. The behavioural change is one coherent edit across the
spine, the three agent-template headers, the four skill templates, and the catalog `fixesAsCommits`
key, landed with its render-assertion updates in a single commit (Phase 1). A second commit (Phase 2)
backs the new invariant, refreshes the `tooling` domain doc and the now-stale stray-branch pitfall,
and flips ADR-0074 to `Implemented`.

Marker names (`apply-fixes-commit`, `re-review-loop`) are kept stable (only their bodies change) so
`skill-section-parity` needs no `internal/catalog/standard.go` `Sections` edit. All templated changes
re-render across **both** the `.claude/` and `.cursor/` trees via `./x sync`.
