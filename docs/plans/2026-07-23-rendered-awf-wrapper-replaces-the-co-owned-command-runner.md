# Plan: Rendered awf wrapper replaces the co-owned command runner

## Goal

Implement ADR-0156: the `runner` singleton renders a pure awf wrapper `awf` at the repo root
(pinned-then-PATH resolution, `awfInvokeCmd` override, default-on), the per-verb forwarding
machinery is deleted, unresolvable hook commands become sync/check errors, and both awf-the-repo
and `examples/sundial` adopt the new split (rendered `./awf` for awf verbs, hand-written `./x`
for project verbs). Non-goals: no change to the generic in-place-section primitive (ADR-0100),
the bootstrap/hooks singletons' own contracts, or the Pi runtime.

## Architecture summary

Four phases, one commit each, matching the ADR's State changes declaration order exactly (V2
batches consume operations in declaration order):

1. **Prune backup** (op: add `runner-prune-backup`): the lock prune backs up a removed co-owned
   runner instead of deleting it. Lands first so every later sync that prunes an `x` dogfoods it.
2. **Core reshape** (ops 2-12): new wrapper template, render unit repath `x` -> `awf`, clispec
   forwarding machinery removed, `awfInvokeCmd` var, default-on seeding (scaffold + migration,
   schema generation 18), awf-the-repo and sundial adopt, core claims and doc prose move.
   This phase is one commit by necessity: `./x check` re-renders the example adopter on every
   gate run, so the template swap and both adoptions cannot be separated without a red gate.
3. **Runner slimming** (op 13): awf's hand-maintained `./x` sheds every awf-verb forwarding arm;
   `commitGateCmd`/`proseGateCmd` flip to `./awf` forms; the dashboard-commands claim rewords.
4. **Hooks and validation** (ops 14-15, final batch): hook payloads drop their inline shims and
   resolve through `./awf`/`awf` via a new `runnerEnabled` render-data flag, the two validation
   error rules land, and the ADR flips to Implemented with the plan and changelog.

Every phase closes with the invariant sequence: stage the complete transaction, run
`go run ./cmd/awf check --staged`, then `./x gate`, then commit. Each phase's ADR
status-history append (Implementing/Applied events with the frozen content digest) follows
`awf-adr-lifecycle`; the staged check validates the claim-transition handshake. Another effort
(ADR-0155 and its plan) concurrently edits skill templates; take a fresh `git status` before
staging and commit by explicit pathspec only.

  renames, remove the original after a successful backup; a backup failure is an error, never a

  `runner/x.tmpl` template id triggers it).
