# Plan: Context and topic output follow-ups and claim splits

## Goal

Fix the three deferred usability findings from the 2026-07-22 context/topic analysis: a glob
query to `awf context` no longer yields misleading domain/topic attribution (F2), collapsed
unowned directories in `awf context --uncovered` carry counts that show they are not wholly
unowned (F3), and the concise text rendering hoists the repeated domain-selector block once
per domain (F4a); finally, apply ADR-0151's meaning-preserving claim splits. Non-goals: glob
expansion support, changed collapse semantics, JSON-shape changes for `awf context`
(the `--uncovered` JSON shape does change), any new doctor heuristic, and the Pi-area
mega-claims excluded by ADR-0151.

## Architecture summary

Phases 1-3 are independent behavioral/presentation changes in `internal/project` (context
assembly, uncovered assembly) and `cmd/awf` (text rendering), each with its own tests,
changelog entry, and doc currency. Phase 4 is ADR-0151's direct Implemented transaction:
claim prose re-authoring in two authored topic parts, proof-marker repointing, one new proof
marker, re-render, and the ADR + plan status flips.
