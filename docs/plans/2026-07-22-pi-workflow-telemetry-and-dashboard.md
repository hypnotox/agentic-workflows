# Plan: Pi Workflow Telemetry and Dashboard

## Goal

Implement [ADR-0146](../decisions/0146-pi-workflow-telemetry-and-dashboard.md) so awf ships a privacy-minimal, trajectory-aware workflow ledger, canonical metrics and doctor surfaces, and a generated Pi dashboard spanning parent-linked sessions. Non-goals are conversational tracing, inferred effort or phase identity, automatic reconciliation, an opaque health score, a background daemon, and a new standalone privacy or generated-artifact document.

## Architecture summary

The implementation uses four checked V2 application batches. The first activates the complete tracked `workflowTelemetry` configuration and schema migration. The second adds the normative protocol plus safe append-only storage, lifecycle, trajectories, privacy enforcement, retention, and the minimal `awf metrics protocol` and lifecycle entrypoints needed to keep all production code reachable. The third adds selectors, projections, aggregation, exact and heuristic diagnosis, and the complete `awf metrics` and `awf doctor` command surfaces. The final batch is intentionally coupled: it adds descriptor-attributed Pi outputs and the dynamic-resident metrics singleton, extends the existing subagent and handoff contracts, ships the dashboard writer/tools/widget/overlay, updates all adopter and project documentation, applies the remaining rendering operations, and co-flips ADR-0146 and this plan to Implemented. Go remains the authority for protocol interpretation, historical aggregation, and diagnosis; TypeScript validates and appends conforming events and refreshes canonical JSON only at controlled boundaries.

## Phase 2: Add the protocol, ledger, lifecycle, trajectories, and retention

- [ ] **Task 2.1: Define one normative protocol descriptor and Go validation model.** Create `internal/telemetry/protocol.json` as the machine authority and embed/parse it in `protocol.go`. The descriptor uses protocol major 1/minor 0; maximum lengths of 128 UTF-8 bytes for opaque IDs, event/idempotency/observation IDs, model names, tool names, and category values; 256 bytes for the repository-relative checkpoint identifier; and rejects empty values, path separators, `.` and `..` for identifiers used as path components. It declares these closed routes: `direct`, `adr`, `plan`, `adr-plan`, `bugfix`, `investigation-only`; these closed phases: `investigation`, `brainstorming`, `adr-authoring`, `adr-review`, `planning`, `plan-review`, `adr-plan-resync`, `implementation`, `implementation-review`, `retrospective`; activities `debugging`, `tdd`, `exploration`, `inline-execution`, `subagent-driven-development`; terminal outcomes `completed`, `abandoned`; association origins `created`, `handoff`, `manual`, `repair`; waiver reason codes exactly those named by ADR-0146's rule table; and event kinds `effort_created`, `session_associated`, `session_detached`, `route_selected`, `route_changed`, `phase_started`, `phase_finished`, `trajectory_started`, `trajectory_resumed`, `trajectory_closed`, `trajectory_forked`, `effort_completed`, `effort_abandoned`, `effort_reopened`, `finding_waived`, `repair_applied`, `usage_observed`, `tool_observed`, `shell_observed`, `compaction_observed`, `handoff_observed`, `subagent_observed`, `session_started`, and `session_ended`.

  The descriptor's exact envelope is `{version:{major:uint16,minor:uint16}, eventId:string, idempotencyKey?:string, observationId?:string, effortId:string, sessionId:string, trajectoryId?:string, parentTrajectoryId?:string, piAnchorId?:string, forkAnchorId?:string, timestamp:string, kind:EventKind, predecessors:string[], payload:Payload}` with `additionalProperties:false`. Exactly one of `idempotencyKey` or `observationId` is required: lifecycle kinds require the former and passive kinds require the latter. `predecessors` contains unique event IDs. `timestamp` is RFC3339Nano. Compatible-minor readers preserve unknown optional top-level fields in `extensions:map[string]json.RawMessage`; an unsupported major or an unknown required kind/payload is rejected.

  For protocol minor 0, the envelope and every payload use `additionalProperties:false`. For a same-major event with `minor > 0`, first validate every known required field and known field value against the minor-0 schema, reject a collision whose known field changes type/meaning, then preserve unknown top-level and payload keys byte-for-byte in separate `EnvelopeExtensions` and `PayloadExtensions` `map[string]json.RawMessage` values; projection ignores those maps. A same-major older/equal minor never admits an unknown key. An unsupported major, unknown discriminator, or absent known required field is rejected. The exact payload discriminator/field table is:

  | Kind | Required payload fields | Optional payload fields |
  |---|---|---|
  | `effort_created` | `checkpointId`, `creationMode` (`independent` or `derived`) | `originEffortId`, `originTrajectoryId`, `originAnchorId` (all three required together only for `derived`; all three forbidden for `independent`) |
  | `session_associated` | `associationOrigin`, `trajectoryId` | `handoffEventId` |
  | `session_detached` | `reason` (`manual`, `pre-association-anchor`, or `repair`) | none |
  | `route_selected`, `route_changed` | `route` | none |
  | `phase_started` | `phase` | `activity`, `implementationMode` |
  | `phase_finished` | `phase`, `startEventId` | `outcome` |
  | `trajectory_started`, `trajectory_resumed`, `trajectory_closed` | `trajectoryId`, `anchorId` | `reason` |
  | `trajectory_forked` | `trajectoryId`, `parentTrajectoryId`, `forkAnchorId` | none |
  | `effort_completed`, `effort_abandoned` | `terminalEpoch` | none |
  | `effort_reopened` | `terminalEpoch`, `trajectoryId`, `anchorId` | none |
  | `finding_waived` | `ruleCode`, `scope`, `evidenceIds`, `reasonCode` | none |
  | `repair_applied` | `proposalKind`, `sourceEventIds`, `replacement` | none; `proposalKind` is `supersede-event`, `correct-phase`, `correct-association`, or `correct-trajectory`; `replacement` is `{eventKind, payload}` where `eventKind` is one lifecycle event kind and `payload` is exactly that kind's closed descriptor payload, never prose |
  | `usage_observed` | `model`, `inputTokens`, `outputTokens`, `cacheReadTokens`, `cacheWriteTokens`, `costUsd`, `durationMs` | `phase`, `activity` |
  | `tool_observed` | `tool`, `outcome`, `durationMs` | `errorCategory` |
  | `shell_observed` | `classification` (`gate` or `unclassified`), `outcome` | `gateMode` (`standard` or `full`) only when classified |
  | `compaction_observed` | `count` | none |
  | `handoff_observed` | `outcome`, `targetSessionId` | `durationMs`, `errorCategory` |
  | `subagent_observed` | `role`, `requestedModel`, `resolvedModel`, `thinkingLevel`, `queueDurationMs`, `runDurationMs`, `inputTokens`, `outputTokens`, `cacheReadTokens`, `cacheWriteTokens`, `costUsd`, `outcome`, `stopReason`, `toolCount`, `toolFailureCount` | `errorCategory` |
  | `session_started`, `session_ended` | `outcome` | `durationMs`, `errorCategory` |

  Every minor-0 payload has `additionalProperties:false`; the same-major newer-minor extension rule above is the only exception. Bounded producer categories are `success`, `failure`, `canceled`, `timeout`, `unavailable`, and `unknown` for outcome; `complete`, `length`, `tool`, `error`, `canceled`, and `unknown` for stop reason; and `filesystem`, `protocol`, `model`, `tool`, `shell`, `gate`, `handoff`, `subagent`, and `unknown` for error category. Payload declarations explicitly forbid prompts, assistant text, tasks, tool arguments/previews, raw shell strings, command output, stderr, free-form waiver prose, and repository paths other than `checkpointId`.
