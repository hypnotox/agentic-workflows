# Plan: Pi Workflow Telemetry and Dashboard

## Goal

Implement [ADR-0146](../decisions/0146-pi-workflow-telemetry-and-dashboard.md) so awf ships a privacy-minimal, trajectory-aware workflow ledger, canonical metrics and doctor surfaces, and a generated Pi dashboard spanning parent-linked sessions. Non-goals are conversational tracing, inferred effort or phase identity, automatic reconciliation, an opaque health score, a background daemon, and a new standalone privacy or generated-artifact document.

## Architecture summary

The implementation uses four checked V2 application batches. The first activates the complete tracked `workflowTelemetry` configuration and schema migration. The second adds the normative protocol plus safe append-only storage, lifecycle, trajectories, privacy enforcement, retention, and the minimal `awf metrics protocol` and lifecycle entrypoints needed to keep all production code reachable. The third adds selectors, projections, aggregation, exact and heuristic diagnosis, and the complete `awf metrics` and `awf doctor` command surfaces. The final batch is intentionally coupled: it adds descriptor-attributed Pi outputs and the dynamic-resident metrics singleton, extends the existing subagent and handoff contracts, ships the dashboard writer/tools/widget/overlay, updates all adopter and project documentation, applies the remaining rendering operations, and co-flips ADR-0146 and this plan to Implemented. Go remains the authority for protocol interpretation, historical aggregation, and diagnosis; TypeScript validates and appends conforming events and refreshes canonical JSON only at controlled boundaries.
