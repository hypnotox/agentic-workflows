# ADR-0015: In-File Provenance for Rendered Output and Convention-Only Section Overrides

## Context

`awf sync` renders the `.claude/awf/` config tree (templates + convention parts + config) into
finished output: docs, skills, agents, hooks, `AGENTS.md`, and domain docs. Section markers are
consumed during assembly (`internal/render/section.go`), so a rendered file carries **no**
back-pointer to its source. ADR-0001 made this deliberate: "The rendered output must contain
**no** generator metadata in its body; provenance lives only in `awf.lock`" (Context), backed by
its Invariants bullet "Rendered skill/agent bodies contain **zero** `awf` markers or provenance
metadata," and enforced by `internal/project/spine_test.go` (`assertNoLeaks`),
`project_test.go:78`, and `render_test.go:36`. That choice optimised for clean, publication-safe
output and for skill authors reading raw templates.

The primary consumer has shifted. AI agents now read rendered files directly, and an agent that
decides to change one cannot tell from the file in its context that (a) it is generated, (b) which
source file to edit, or (c) the precedence rules. Provenance in `awf.lock` is out-of-band: an
agent holding the rendered file would have to know the lock exists, open it, and reverse-resolve
the section, defeating the point. The fix that helps an agent is the one that travels *with* the
content it is editing.

The config tree (ADR-0009) offers two ways to replace a section body: a **convention part** at
`.claude/awf/<kind>/parts/<target>/<section>.md`, and an explicit `replaceWith:` pointer in a
sidecar. A grounding sweep established:

- `replaceWith` is used **nowhere** in this repo's config tree; the only sidecar `sections:` entry
  is `skills/refactor-coupling-audit.yaml`, which uses `drop:` only.
- Convention parts are *already* implemented internally by setting `SectionOverride.ReplaceWith`
  to the conventional path (`internal/project/project.go:224`); the config-facing field is an
  unused escape hatch.
- Section identity and the winning source are both known at `Assemble` time
  (`internal/render/render.go`), and the per-target `ConfigHash` hashes inputs, not rendered
  output (`project.go:454`), so deterministic injection does not disturb drift detection.
- Frontmatter is split in `internal/frontmatter` (ADR-0006); a banner placed before a skill's
  leading `---` would break frontmatter parsing.
- Hooks (`templates/hooks/*.tmpl`) are shell scripts with no section markers; domain docs
  (`templates/domains/domain.md.tmpl`) and `AGENTS.md` participate in the same section mechanism.

Removing the unused `replaceWith` pointer makes the conventional part path the single,
deterministic edit target for every section, which is what lets an in-file pointer name that
target unambiguously. The two changes are therefore bundled: provenance is the goal,
convention-only overrides is its enabling simplification.

## Decision

1. **Provenance banner.** Every rendered file begins with a generated-by banner:
   `GENERATED by awf: do not edit; change .claude/awf/ and run \`awf sync\``. Comment syntax is
   kind-aware: HTML comments (`<!-- ... -->`) for markdown targets (docs, skills, agents, domains,
   `AGENTS.md`); `#` line comments for shell hooks.

2. **Frontmatter-aware banner placement.** For targets carrying YAML frontmatter (skills,
   agents), the banner is emitted immediately *after* the closing `---`, never before it, so
   frontmatter parsing (ADR-0006) is preserved. For shell hooks the banner follows the `#!`
   shebang line (which must remain line 1 for the script to stay executable), never before it.
   For all other targets it is the first line.

3. **Per-section edit pointer.** Each rendered, non-dropped section is immediately preceded by an
   `awf:edit` comment naming the winning source and the deterministic edit target:
   - section on the template default â†’ `awf:edit <section>: default; create <conventional part path> to override`
   - section overridden by a convention part â†’ `awf:edit <section>: from <conventional part path>`
   - dropped section â†’ no output (nothing to annotate).

4. **Convention-only overrides.** Remove the config-facing `replaceWith` field from
   `config.SectionOverride`; `drop` is retained. A section body is replaced *only* by a convention
   part file existing at its conventional path. Per-section precedence collapses to
   `drop > convention part > template default`. Because the convention part can no longer ride the
   removed `replaceWith` field into `render.Assemble` (today the project layer injects a synthetic
   `SectionOverride{ReplaceWith: <convention path>}` at `project.go:224` and `Assemble` renders it
   via its `replaceWith` branch at `render.go:27`), the project layer instead resolves the
   convention part and passes it to `Assemble` through a dedicated, non-`replaceWith` channel, so
   `Assemble`'s overlay switch collapses to `drop > convention part > default`. This overrides
   **ADR-0001 Decision item 2** (`supersedes: ADR-0001#2`; the `replaceWith` overlay step) and the precedence documented by
   **ADR-0009** (its Decision item 4, `refines: ADR-0009#4`, and `inv: parts-convention`, `supersedes-invariant: ADR-0009#parts-convention`).

5. **Migration.** `awf upgrade` (ADR-0010) bumps the config-tree schema version. A sidecar still
   carrying a `replaceWith:` is migrated by relocating the referenced part to the section's
   conventional path (`.claude/awf/<kind>/parts/<target>/<section>.md`, or
   `.claude/awf/parts/agents-doc/<section>.md` for the agents-doc singleton (ADR-0009 item 7)) and
   dropping the field. If the conventional path is already occupied by a *different* file (an
   identical file is a no-op), or the referenced part is missing, `upgrade` fails with an
   actionable message rather than silently overwriting or dropping content.

6. **Scope of supersedence.** This ADR overrides **ADR-0001 Invariants bullet 1** ("zero `awf`
   markers or provenance metadata"), narrowing it to permit the banner and `awf:edit` pointers,
   and its Context claim that "provenance lives only in `awf.lock`"; plus **ADR-0001 Decision item
   2** (`supersedes: ADR-0001#2`; the `replaceWith` overlay step, per Decision item 4). It also overrides **ADR-0009 Decision
   item 4** (`refines: ADR-0009#4`) and its `inv: parts-convention` (`supersedes-invariant: ADR-0009#parts-convention`), collapsing that ADR's four-tier precedence
   (`drop > explicit replaceWith > convention part > template default`) to the three-tier
   `drop > convention part > template default`. The rest of ADR-0001 remains in force: Engine B,
   `missingkey=zero`, and bullet 4's rule that `awf:section`/`awf:end` markers never appear in
   rendered output. Both ADR-0001 and ADR-0009 keep their existing status (`Accepted` and
   `Implemented` respectively); this is partial-item supersedence recorded via `related`, not a
   full replacement. Because ADR-0009 is `Implemented`, its `parts-convention` backing test is
   updated in lockstep when this ADR reaches `Implemented`.

## Invariants

- `invariant: provenance-banner`: every rendered file carries the awf generated-by banner as its first
  line, except where it must follow a leading construct: after the closing frontmatter delimiter
  for targets that carry frontmatter, and after the `#!` shebang line for shell hooks.
- `invariant: section-edit-pointer`: every rendered, non-dropped section is immediately preceded by an
  `awf:edit` pointer naming its conventional part path.
- `invariant: no-section-marker-leak`: `awf:section` / `awf:end` markers never appear in rendered
  output; `awf:edit` pointers and the banner are the only awf markers a rendered file may carry.
  (Restates ADR-0001 Invariants bullet 4's section-marker ban, which remains in force, and narrows
  bullet 1's "zero `awf` markers" clause to permit the banner and `awf:edit` pointers.)
- `invariant: no-replacewith`: `config.SectionOverride` exposes no `replaceWith` field; the sole
  section-replacement mechanism is a convention part.

## Consequences

Easier:
- An agent edits the correct source from the file it already has in context: no out-of-band lock
  lookup, no precedence reasoning, no extra turn.
- The conventional part path is the single deterministic edit target, so the pointer is
  unambiguous and default sections advertise exactly where to create an override.
- Less config surface: `replaceWith` and the explicit-replace branch in `Assemble` are gone.
  `assertNoLeaks` (`spine_test.go:53`) already rejects only `awf:section`/`awf:end`, so it
  tolerates `awf:edit`/banner today; it gains *positive* assertions that the banner and per-section
  pointers are present rather than needing to be loosened.

Harder:
- Rendered files gain comment surface (invisible in rendered markdown, present in source). awf's
  fingerprint now appears in every consumer project's rendered output.
- Every golden/render test shifts to include the banner and pointers (mechanical but wide).
- Banner injection must be frontmatter-aware; per-section injection must thread `kind`/`target`
  through to resolve the conventional path.

Ruled out:
- Out-of-body provenance (`awf.lock` only) as the *sole* mechanism: invisible to an agent reading
  the rendered file.
- Round-trip reverse-sync (edit the rendered file, awf writes the delta back): attribution is
  ambiguous and it breaks the never-hand-edit-a-rendered-file invariant.

Doc-currency obligations the implementing commit(s) must satisfy:
- `docs/architecture.md` updates the per-section precedence it documents (currently
  `drop > explicit replaceWith > convention part > template default` at the layout and render-flow
  sections) to `drop > convention part > template default`, removes the `replaceWith` references,
  and notes the in-file banner / `awf:edit` provenance now present in rendered output.
- ADR-0009's `inv: parts-convention` backing test is updated to the new precedence in the same
  change that flips this ADR to `Implemented`.
- When this ADR flips to `Accepted`/`Implemented`, the same commit regenerates
  `docs/decisions/ACTIVE.md` via `./x sync`. No `docs/decisions/README.md` index row is owed (the
  README is a how-to guide; `ACTIVE.md` is the generated index, ADR-0005).

## Alternatives Considered

| Alternative | Why not chosen |
|---|---|
| Resolver command only (`awf where <file>#<section>`) | Out-of-context: the agent must know the tool exists and spend a turn; loses the in-context locality that is the entire point. |
| Round-trip reverse-sync | Attributing an arbitrary rendered-file edit back to a section/part is ambiguous (default text, cross-section edits, reformatting) and breaks the drift oracle. |
| Banner only, no per-section pointers | The agent still has to hunt for which section's source to edit; per-section pointers are the locality win. |
| Keep `replaceWith`, annotate its path | Retains unused config surface and a non-deterministic edit target, making the pointer ambiguous. |
